# learn-express

## 프로젝트 개요

Express와 Multer를 이용해 브라우저에서 이미지 한 개를 서버 디스크로 업로드하는 과정을 학습하는 Node.js 프로젝트입니다. 루트 안내 화면과 multipart 업로드 폼을 제공하며 업로드 형식, 크기와 서버 생성 파일명을 제한합니다.

## 구현된 라우트

| 메서드 | 경로 | 기능 |
|---|---|---|
| `GET` | `/` | `index.html` 학습 안내 화면 반환 |
| `GET` | `/upload` | `multipart.html` 업로드 폼 반환 |
| `POST` | `/upload` | `image` 필드의 파일 한 개를 `uploads/`에 저장하고 `ok` 반환 |

업로드 폼의 `title` 입력은 현재 서버에서 읽거나 저장하지 않습니다.

## 업로드 정책

- 허용 MIME 형식: GIF, JPEG, PNG, WebP
- 최대 파일 크기: 5 MiB
- multipart 필드 이름: `image`
- 저장 위치: 저장소 루트 기준 `uploads/`
- 파일명: 원본 이름 대신 시각과 난수 값을 조합한 서버 생성 이름

서버는 시작할 때 `uploads/`가 없으면 생성합니다. 이 디렉터리는 Git에서 제외되며 정적 파일 라우트로 공개되지 않습니다. 업로드 목록, 다운로드, 삭제, 만료, 용량 관리와 파일 메타데이터 저장 기능도 없습니다.

MIME 값은 클라이언트가 보낸 `file.mimetype` 문자열만 검사합니다. 실제 파일 바이트의 시그니처나 콘텐츠를 확인하지 않으므로 확장자·MIME 위장과 악성 파일을 막는 보안 검사로 간주해서는 안 됩니다.

## 기술 스택

- Node.js
- Express 4.19.2
- Multer 1.4.5 LTS
- express-session 1.18.0
- cookie-parser 1.4.6
- Morgan
- dotenv
- Nodemon
- Node 내장 테스트 러너

`package-lock.json`이 의존성 트리를 고정하지만 지원 Node.js 버전은 `package.json`에 선언되어 있지 않습니다.

## 프로젝트 구조

```text
.
├── app.js                    # Express 앱, 미들웨어와 업로드 라우트
├── upload-policy.js          # 허용 MIME과 서버 파일명 정책
├── index.html                # 루트 안내 화면
├── multipart.html            # 업로드 폼
├── test/
│   └── upload-policy.test.js
├── package.json
└── package-lock.json
```

`app_old1.js`는 이전 학습 코드이며 현재 `npm start` 진입점이 아닙니다.

## 설치 및 실행

저장소 루트에서 잠금 파일 기준으로 의존성을 설치합니다.

```bash
npm ci
npm start
```

`npm start`는 `nodemon app`을 실행합니다. 기본 주소는 `http://127.0.0.1:3000/`이며 개발 중 파일 변경을 감지해 서버를 다시 시작합니다. 별도의 운영 실행·배포 스크립트는 없습니다.

## 환경 변수

코드는 `.env`를 읽으며 다음 이름을 참조합니다.

| 이름 | 용도 |
|---|---|
| `PORT` | 서버 포트, 미설정 시 `3000` |
| `COOKIE_SECRET` | 서명 쿠키와 세션 비밀값 |
| `NODE_ENV` | `production`일 때 세션 쿠키의 `secure` 활성화 |

실제 환경 변수 값은 README, 이슈, 로그와 커밋에 기록하지 마세요.

## 추적된 `.env` 보안 조치

> **현재 상태에서 저장소와 서버를 공개하거나 배포하지 마세요.** Git이 추적하는 `.env`와 전체 이력에 `COOKIE_SECRET`으로 사용될 수 있는 값이 남아 있습니다. 실제 값은 이 README에 기록하지 않습니다.

안전한 공개·배포 전에 다음 조치가 필요합니다.

1. 기존 비밀값을 폐기하고 안전한 새 값으로 교체합니다.
2. 기존 서명 쿠키와 세션을 무효화합니다.
3. `.env`를 Git 추적 대상에서 제거하고 `.gitignore`에 추가합니다.
4. 실제 값이 없는 안전한 예제 환경 파일을 마련합니다.
5. 협업자와 조율하고 승인을 받은 방법으로 Git 이력을 정리합니다.
6. 현재 파일, 전체 이력과 배포 환경에서 비밀값이 제거됐는지 재검증합니다.

## 세션과 데이터 저장

데이터베이스와 외부 API는 사용하지 않습니다. 업로드 파일만 로컬 디스크에 저장합니다.

Express Session은 별도 저장소를 지정하지 않아 기본 `MemoryStore`를 사용합니다. 현재 세션 값을 저장·조회하는 사용자 기능은 없지만, 향후 사용할 경우 서버 재시작 때 세션이 사라지고 여러 프로세스가 세션을 공유하지 못합니다. 기본 MemoryStore는 운영 환경용이 아닙니다.

## 테스트

```bash
npm test
```

`test/upload-policy.test.js`에는 두 테스트와 세 개의 `assert.equal` 검증이 있습니다.

- PNG는 허용하고 HTML MIME은 거부하는지 확인
- 원본 사용자 파일명 대신 예측 가능한 테스트 입력으로 서버 파일명을 생성하는지 확인

라우트, multipart 처리, 5 MiB 제한, GIF·WebP, 실제 파일 저장, 오류 응답, 세션과 보안은 테스트하지 않습니다.

## 보안 및 운영 제한

- 인증과 사용자별 권한이 없습니다.
- CSRF 보호가 없습니다.
- 요청 속도와 사용자별 업로드 용량 제한이 없습니다.
- MIME 문자열 외에 파일 시그니처·악성 콘텐츠 검사가 없습니다.
- 업로드 파일 정리, 격리, 백업과 보존 정책이 없습니다.
- 서버 생성 파일명은 충돌 방지용 저장 이름이며 보안 토큰이 아닙니다.
- 업로드 파일 정보가 콘솔 로그에 남습니다.
- 잘못된 형식과 크기 초과를 포함한 업로드 오류가 일반 `500` 응답으로 처리됩니다.
- `public/` 정적 디렉터리를 등록하지만 현재 저장소에는 해당 디렉터리가 없습니다.
- HTTPS, 프록시, 운영 프로세스, 관측과 배포 구성이 없습니다.

Git 이력에는 과거 `node_modules`가 추적된 커밋이 있습니다. 현재는 디렉터리가 없고 `.gitignore`에 포함되지만 공개 전에는 이력 크기와 포함된 제3자 파일·라이선스를 검토해야 합니다. 이력 정리가 필요하면 협업자 승인 후 수행해야 합니다.
